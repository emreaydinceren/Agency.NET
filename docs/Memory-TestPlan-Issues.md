### TI-1 — No single DI entry point boots all four hosted services

- **Where:** `Agency.Memory.Distiller/DependencyInjection/MemoryServiceCollectionExtensions.cs` (`AddAgencyMemory`).
- **Observation:** `Memory-TestPlan.md` §2.1 says the fixture boots "all four hosted services" via `AddAgencyMemory(...)`. The current `AddAgencyMemory` wires the distiller background service + inactivity timer only. The consolidator (`AddAgencyConsolidator` in `Agency.Memory.Consolidator`) and hygiene sweeper register separately, and the backing store/embedder/LLM adapter are not registered by `AddAgencyMemory` at all.
- **Impact:** D0.1 must compose multiple DI extensions to stand up the full host. Not a bug — a wiring question.
- **Proposed handling:** Have SA-T0 compose the available extensions in the fixture and document the exact call sequence. If a unified `AddAgencyMemory` overload is the spec intent, file a spec-gap follow-up rather than expanding scope inside a test. **Do not** modify production DI as part of test authoring (Memory-TestPlan.md §4, surgical-changes rule).
- **Status:** **Resolved** — fixture composes the available DI extensions; no production change required for the distiller/consolidator/timer trio.

### TI-2 — Hygiene sweeper has no test seam (blocks Group 4)

- **Where:** `Agency.Memory.Hygiene/HygieneSweeperBackgroundService.cs` (`internal sealed`); `Agency.Memory.Hygiene/AssemblyInfo.cs` (no `InternalsVisibleTo` for the functional test project); no public `AddAgencyHygiene()` DI extension exists.
- **Observation:** §2.1 says the fixture boots "all four hosted services"; §3-Group 4 and §6.6 require the sweep to be triggered by `PeriodicTimer.Tick`. Neither is reachable from the test project today.
- **Why it is not mere scope-drift:** §10 of the spec lists all four hosted services as the deployed topology, so a public seam to register the sweeper is part of the *specified* surface — not a gratuitous "improvement" forbidden by Memory-TestPlan.md §4. SA-G already applied `InternalsVisibleTo("Agency.Memory.Functional.Test")` to the other **four** memory assemblies (Distiller, Retrieval, Common, Consolidator — see Memory-Tracker.md SA-G row); Hygiene was simply missed because no G.* test exercised it.
- **Manager decision (2026-05-28):** Authorise a **minimal** production seam, treated as completing Workstream F's wiring (not a test-time improvement):
  1. Add `[assembly: InternalsVisibleTo("Agency.Memory.Functional.Test")]` to `Agency.Memory.Hygiene/AssemblyInfo.cs`, **and**
  2. Add a public `AddAgencyHygiene(...)` DI extension in `Agency.Memory.Hygiene` that registers `HygieneSweeperBackgroundService` as an `IHostedService` and accepts an injected `TimeProvider` (so `TimeShim.Provider` can be substituted in tests).
  This is assigned to a dedicated sub-agent (**SA-Hyg**) and must land + build 0/0 **before** the D4 authoring agent starts. The weaker `DeleteWhere*Async`-direct workaround SA-T0 proposed is rejected because it tests the delete query, not the §6.6 sweep trigger.
- **Spec-gap follow-up:** If the project owner prefers the sweeper stay un-exposed, the alternative is to mark Group 4 as driving the store directly and annotate the §6.6 anchor as partially covered. Flagged here for visibility; default path is the seam above.
- **Status:** **Resolved** — SA-Hyg implemented both items; build 0/0; manager-verified. **D4 carry-over:** register the sweeper via `services.AddSingleton<TimeProvider>(timeShim.Provider); services.AddAgencyHygiene(o => o.HygieneSchedule = <small>);` and/or drive a single sweep through the now-visible `internal HygieneSweeperBackgroundService.RunOnceAsync(ct)`. The D0.1 fixture does **not** yet register hygiene — the D4 author must extend the fixture host or build a D4-local host that adds `AddAgencyHygiene`. Sweep interval option: `MemoryOptions.HygieneSchedule` (default 24h); jitter is ±15 min via `ApplyJitter`.
